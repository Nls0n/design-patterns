## Архитектурные паттерны

Реализация разделена на функциональные уровни, каждый из которых интегрирует специфические паттерны проектирования:

### 1. Порождающие паттерны 
*   **Singleton (Одиночка):** Гарантирует создание единственного экземпляра конфигурации базы данных. Реализовано через метаклассы.
*   **Abstract Factory (Абстрактная фабрика):** Инкапсулирует логику создания семейств ингредиентов (тесто, соус) для различных типов кухни.
*   **Decorator (Декоратор):** Позволяет динамически изменять стоимость и состав базового продукта (например, добавление сыра) без модификации исходного класса.

### 2. Структурные паттерны 
*   **Adapter (Адаптер):** Обеспечивает совместимость современной системы учета с устаревшими платежными шлюзами 
*   **Composite (Компоновщик):** Формирует иерархическую структуру меню , позволяя обрабатывать их единообразно.
*   **Proxy (Заместитель):** Реализует контроль доступа и ленивую инициализацию тяжелых компонентов пользовательского интерфейса.

### 3. Поведенческие паттерны (Behavioral)
*   **State (Состояние):** Управляет жизненным циклом заказа, делегируя логику изменения статусов специализированным классам.
*   **Strategy (Стратегия):** Позволяет переключать алгоритмы ценообразования (например, применение скидок) в рантайме.
*   **Template Method (Шаблонный метод):** Определяет базовый алгоритм обработки заказа, позволяя наследникам переопределять шаги валидации и оплаты.
*   **Observer (Наблюдатель):** Обеспечивает слабую связанность модулей, оповещая заинтересованные сервисы об изменениях в процессе приготовления.
*   **Command (Команда):** Превращает запросы на приготовление в объекты, поддерживая возможность постановки задач в очередь.
*   **Iterator (Итератор):** Предоставляет унифицированный интерфейс для последовательного обхода сложных древовидных структур меню.

##  Математическая модель

В систему интегрирован модуль оптимизации логистики, использующий алгоритм **A* (A-Star)**. Модель предназначена для нахождения кратчайшего пути доставки по графу городских узлов. Эвристическая функция позволяет минимизировать время курьера в пути, обеспечивая эффективность этапа доставки.

##  Развертывание и запуск

Проект полностью автоматизирован для запуска в изолированной среде Docker.

### Команды терминала:
1.  **Запуск проекта:**
    ```bash
    make run
    ```
*Команда выполнит сборку образа, инициализацию PostgreSQL, применение схем БД и наполнение таблиц тестовыми данными (20+ записей).*
2.  ** Остановка проекта:**
    ```bash
    make stop
    ```

##  API Эндпоинты

| Метод | Путь | Описание |
| :--- | :--- | :--- |
| `GET` | `/` | Интерфейс пользователя (защищен через Proxy) |
| `GET` | `/orders` | Реестр заказов из базы данных |
| `GET` | `/patterns/math` | Визуализация кратчайшего пути (A*) |
| `GET` | `/patterns/strategy` | Расчет цены с применением активной стратегии |
| `GET` | `/patterns/composite_iterator` | Вывод структуры меню через итератор |
| `GET` | `/docs` | Автогенерируемая документация Swagger/OpenAPI |
